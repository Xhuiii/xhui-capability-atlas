# xhui-capability-atlas

A sustainable, interactive, and visual personal skill simulator atlas for portfolio, career transition, self-growth, creative experiments, and long-term asset accumulation.

一个可持续生长、可交互、可视化的个人技能模拟器地图，用于作品集、职业转向、自我成长、创意实验和长期个人资产沉淀。

---

## 1. Purpose / 项目目的

This repository is used to build and maintain my personal skill simulator system.

这个仓库用于搭建和维护我的个人技能模拟器系统。

It is not only a portfolio, a resume archive, or a static capability map. It is a long-term, structured, and interactive system that collects my real projects, skills, methods, evidence, reflections, creative ideas, what-if assumptions, and future directions.

它不是普通作品集，也不是静态简历库。它是一个长期、结构化、可交互的系统，用来持续收纳我的现实项目、技能、方法论、证据链、复盘、创意想法、what-if 设定和未来方向。

It helps me organize both the abilities that have already been proved and the possible abilities that may grow in different assumptions, contexts, timelines, and creative experiments.

它帮助我把“已经被证明的能力”和“在不同假设、不同语境、不同时间线、不同创意实验中可能长出来的能力”都整理成可持续生长的个人资产。

It serves portfolio building and career transition, but it also serves long-term personal growth, product experiments, content expression, aesthetic exploration, and future commercial imagination.

它既服务于作品集和职业转向，也服务于更长期的个人成长、产品实验、内容表达、审美探索和未来商业想象。

## 2. Core Logic / 核心逻辑

The whole repository is organized around one main layer and five input layers.

整个仓库围绕“一个主层 + 五个输入层”组织。

- P0 is the main skill simulator atlas.
- P1-P5 are different input systems that continuously feed P0.
- Issues are used to turn ideas into concrete tasks.
- Milestones are used to control execution stages.
- Safety rules define what can and cannot be stored.
- Update logs preserve how this project evolves over time.

中文对应：

- P0 是主技能模拟器地图。
- P1-P5 是持续向 P0 输入内容的不同系统。
- Issues 用来把想法拆成具体任务。
- Milestones 用来控制阶段推进。
- Safety rules 用来定义哪些内容可以存、哪些内容不能存。
- Update Log 用来保留项目定义和思考过程的演进。

简单来说：

P1 Agent 工作台、P2 AI 产品实验、P3 内容与个人品牌、P4 设计师的店、P5 运动健康与自我成长，会持续向 P0 输入、反哺和沉淀，最终汇聚到 P0 Personal Skill Simulator Atlas / 个人技能模拟器地图。

P0 is not an isolated portfolio. It continuously receives inputs from P1-P5 and transforms them into capability cards, project cards, what-if expression cards, evidence chains, timeline records, skill simulation scenarios, and future direction maps.

P0 不是孤立的作品集。它会持续接收 P1-P5 的输入，并把这些输入转化成能力卡、项目卡、What-if 表达卡、证据链、时间线、技能模拟场景和未来方向地图。

## 3. P0 — Personal Skill Simulator Atlas / 个人技能模拟器地图

P0 is the main product and the core layer of this repository.

P0 是这个仓库的主产品，也是整个系统的核心层。

It shows proven skills from real projects, visible skills from past work, and possible skills revealed through what-if assumptions, speculative thinking, magical ideas, product imagination, playful systems, content experiments, and my personal way of seeing the world.

它不只展示现实项目中已经被证明的技能、过往工作中可见的技能，也展示我通过 what-if 设定、思辨设计、魔法版脑洞、产品想象、玩乐化系统、内容实验和个人观察方式显影出来的潜在技能。

P0 asks not only:

P0 不只问：

- What skills have I already shown?
- What capabilities have my projects already proved?

中文对应：

- 我已经展示过什么技能？
- 我的项目已经证明了什么能力？

It also asks:

它还会问：

- How do I see the world?
- What assumptions do I naturally create?
- If I could see the shapes of emotions, what would I make?
- If animals could speak, what would they need?
- If a city could sense loneliness, what would change?
- What hidden skills are revealed through these what-if expressions?

中文对应：

- 我如何看待世界？
- 我会自然地产生什么假设？
- 如果我能看见情绪的形状，我会做出什么？
- 如果动物会说话，它们会需要什么？
- 如果一座城市能感知孤独，它会发生什么变化？
- 这些 what-if 表达显影出了我哪些隐藏技能？

P0 treats these what-if ideas not as random fantasies, and not only as concept design. They are skill simulations.

P0 不把这些 what-if 想法当成随机幻想，也不把它们仅仅看作概念设计。它们本质上是技能模拟。

They reveal my ability to observe the world, create assumptions, define invisible needs, imagine alternative users, translate abstract feelings into systems, and turn poetic thinking into products, services, interfaces, stories, rituals, tools, games, content, or other possible forms.

它们能体现我的观察力、假设能力、隐性需求定义能力、另类用户想象能力、抽象情绪产品化能力，以及把诗意感受转化为产品、服务、界面、故事、仪式、工具、游戏、内容或其他形态的能力。

Concept design is only one possible method. A what-if expression may also become a product, a service, a tool, a game, a content series, a commercial idea, an interactive narrative, or something that does not yet have a clear category.

概念设计只是其中一种表达手法。一个 what-if 表达也可能变成产品、服务、工具、游戏、内容系列、商业想象、互动叙事，或者某种暂时还没有明确分类的形态。

## 4. Project Domains / 项目域

This repository contains six project domains.

这个仓库包含六个项目域。

### P0 — Personal Skill Simulator Atlas / 个人技能模拟器地图

**Status: Active mainline / 当前主线**

P0 is the main skill simulator atlas. It collects and transforms real skills, project skills, what-if ideas, speculative thinking, magical ideas, product imagination, content experiments, and self-growth inputs into a visual and interactive personal skill map.

P0 是主技能模拟器地图。它负责把现实技能、项目技能、what-if 想法、思辨设计、魔法版脑洞、产品想象、内容实验和自我成长输入，转化成一张可视化、可交互、可持续生长的个人技能地图。

It is not a normal portfolio or resume archive. It is the core system for simulating both “what skills I already have” and “what skills I may grow.”

它不是普通作品集，也不是简历库，而是用来模拟“我已经有什么能力”和“我还可能长出什么能力”的核心系统。

### P1 — Agent Workbench / Agent 工作台

**Status: Active support system / 当前支撑系统**

P1 builds the personal AI agent workflow powered by Hermes, Discord, GitHub, Flomo, and future coding agents.

P1 搭建由 Hermes、Discord、GitHub、Flomo 和未来 coding agents 支撑的个人 AI 项目经理工作流。

It helps capture ideas, classify them into P0-P5, manage issues, create daily action suggestions, and keep the whole system from becoming chaotic.

它负责接住想法、判断想法属于 P0-P5 哪个项目域、管理 GitHub issues、生成每日行动建议，并防止整个系统变得混乱。

Its purpose is not to create more tasks. Its purpose is to help me reduce noise, clarify priorities, and turn scattered ideas into small executable actions.

它的作用不是制造更多任务，而是帮我减少噪音、明确优先级，并把散落的想法拆成可以执行的最小行动。

P1 provides project management records, idea routing results, daily action suggestions, review notes, and agent skill maps to P0.

P1 会向 P0 输入项目管理记录、想法分流结果、每日行动建议、复盘内容和 agent skill map，让 P0 不只是静态展示，而是可以持续更新。

### P2 — AI Product Lab / AI 产品实验池

**Status: Next track / 近期准备启动**

P2 collects AI product experiments, side-project ideas, small tools, high-freedom game concepts, playful systems, interactive narrative, sandbox-style creation tools, and broader product imagination.

P2 收纳 AI 产品实验、副业想法、小工具、高自由度游戏概念、玩乐化系统、互动叙事、沙盒式创作工具，以及跨度更大的产品想象。

It is not limited to narrow AI tools. It can include AI side-business evaluators, AI portfolio tools, high-freedom game products, pea-like game ideas, interactive narrative systems, sandbox-style creation tools, AI companions or character systems, and product ideas that are not yet easy to categorize.

它不只收纳狭义 AI 工具，也可以包括 AI 副业判断器、AI 作品集工具、高自由度游戏产品、豌豆类游戏产品、互动叙事、沙盒式创作工具、AI 陪伴/角色系统，以及暂时还无法归类的产品想象。

P2 provides product imagination, system design ability, AI-native thinking, prototype ability, and experiment cases to P0.

P2 会向 P0 输入我的产品想象力、系统设计能力、AI-native 思考、原型能力和实验案例。它可以成为未来作品集、内容表达或商业想象的素材来源。

### P3 — Content and Personal Brand / 内容与个人品牌

**Status: Next track / 近期准备启动**

P3 collects content ideas, personal expression, Xiaohongshu topics, AI-generated media, design reflections, and personal brand experiments.

P3 收纳内容选题、个人表达、小红书主题、AI 生成媒体、设计观察、设计复盘和个人品牌实验。

It is not only about posting content. It is a way to train expression ability, aesthetic identity, personal narrative, and public visibility.

它不只是“发内容”，而是用内容反过来训练我的表达能力、审美人格、个人叙事和公众可见度。

P3 provides content assets, expression style, personal voice, audience feedback, and aesthetic identity to P0.

P3 会向 P0 输入内容资产、表达风格、个人声音、受众反馈和审美身份，让 P0 不只是能力地图，也能体现“我是一个怎样的设计师 / 创作者”。

### P4 — Designer Shop Lab / 设计师的店

**Status: Backlog / 暂存**

P4 collects design studio ideas, aesthetic experiments, digital products, commercial concepts, and future productization possibilities.

P4 收纳设计工作室想法、审美实验、数字产品、商业概念和未来产品化可能性。

It is currently kept in backlog to avoid opening too many main directions at once. But it is important because it carries my long-term imagination about how design can become works, products, experiences, or a way of living.

它现在先作为 Backlog 暂存，避免同时开启太多主线方向。但它很重要，因为它承接的是我对“设计如何变成作品、商品、体验或生活方式”的长期想象。

P4 provides commercial imagination, aesthetic experiments, personal IP directions, and future productization possibilities to P0.

P4 会向 P0 输入商业想象、审美实验、个人 IP 方向和未来产品化可能性。

### P5 — Exercise, Health, and Self-Growth / 运动健康与自我成长

**Status: Lightweight next track / 轻量近期推进**

P5 is a lightweight self-growth support system for exercise, energy, routines, habits, and future wearable data.

P5 是轻量的运动健康与自我成长支持系统，用于记录运动、精力、睡眠、习惯、状态变化和未来可能接入的可穿戴数据。

It is not a medical system and should not create anxiety. Its goal is not to make me over-monitor myself, but to help me act more steadily, live with more energy, and support long-term creation and project execution.

它不是医疗系统，也不是焦虑制造器。它的目标不是让我过度监控自己，而是帮我更稳定地行动、更有能量地生活，并支撑长期创作和项目推进。

P5 provides state variables, execution records, habit changes, and self-growth traces to P0.

P5 会向 P0 输入状态变量、行动力记录、习惯变化和自我成长轨迹，让 P0 不只展示能力，也能看到能力背后的能量和长期执行条件。

## 10. Safety Rules / 安全规则

- No company data / 不存放公司资料
- No confidential project details / 不存放内部项目细节
- No private credentials / 不存放私人凭据
- No sensitive personal documents / 不存放敏感个人文件
- No token, API key, auth file, or secret / 不存放 token、API key、auth 文件或任何密钥
- No real colleague names or internal organizational details / 不存放真实同事姓名或内部组织信息
- Only manually provided and desensitized materials / 只使用手动提供且已脱敏的材料
- Agent actions require confirmation / Agent 执行动作前需要确认
- GitHub issues are used for planning and tracking, not for storing secrets / GitHub issues 只用于规划和追踪，不用于保存秘密信息

This repository can only store manually organized and desensitized personal project materials. It must not store company-sensitive information, internal project details, credentials, tokens, private files, real colleague information, or sensitive health information.

这个仓库只能存放手动整理、脱敏后的个人项目内容，不能存放公司敏感资料、内部项目细节、密钥、token、私密文件、真实同事信息或敏感健康资料。

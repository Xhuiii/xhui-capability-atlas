# Capability Cards v0 / Career Capability Layer 职业能力成长索引 v0

本文件属于 P0 / Career Capability Layer / 职业能力层。

它不是 P0 的全部，也不把 P0 缩小成职业总结。P0 的完整定位仍然是 `Personal Skill Simulator Atlas / 个人技能模拟器地图`，会持续容纳真实项目能力、What-if 想象表达、AI 产品实验、内容与个人品牌、设计师的店与审美实验等多个层。

本文件从 2022-2026 年度项目成长路径中抽象职业能力，为后续动态可视化能力地图提供能力节点。它未来可以和 P0 的 What-if Expression Layer / 想象表达层、AI Product Lab Layer / AI 产品实验层、Content & Personal Brand Layer / 内容与个人品牌层、Designer Shop / Studio Layer / 设计师的店与审美实验层发生连接，但不替代它们。

当前内容全部使用脱敏、抽象表达，不包含公司敏感信息、内部数据来源、同事姓名或未公开细节。

## C01 — 复杂业务与系统体验拆解能力 / Complex Business and System UX Decomposition

### 能力编号 / Capability ID

C01

### 所属 P0 层级 / P0 Layer

Career Capability Layer / 职业能力层

### 能力名称 / Capability Name

复杂业务与系统体验拆解能力 / Complex business and system UX decomposition

### 一句话定义 / One-line Definition

能够把复杂业务、系统规则、多角色场景和高约束体验拆解为清晰的用户任务、流程结构、状态逻辑和设计机会。

### 来源年份 / Source Years

2022, 2023, 2025, 2026

### 来源项目阶段 / Source Project Stages

- 2022：B 端 UX 起点，复杂业务理解与结构化设计。
- 2023：全场景协同，跨端链路和场景关系理解。
- 2025：系统基础体验与通信体验过渡。
- 2026：通信竞争力和智能化信息体验深化。

### 证据类型 / Evidence Types

- flow_map
- task_flow
- state_logic
- scenario_matrix
- design_solution
- metrics

### 代表性成果 / Representative Outcomes

- 复杂业务流程和任务链路拆解。
- 多场景、多状态体验方案。
- 通信与智能信息体验中的关键路径设计。
- 可与体验指标和交付结果建立映射的设计证据。

### 可迁移方向 / Transferable Directions

- B2B / SaaS 产品体验
- 系统级产品体验
- AI 工具工作流
- 多角色协作系统
- 复杂场景服务设计

### 可视化呈现方式 / Visualization Method

在 P0 / Career Capability Layer 中作为底层核心能力节点，用“年份 -> 项目阶段 -> 复杂度类型 -> 设计动作 -> 可见结果”的链路展示。

### 后续更新方式 / How to Keep Updating

每新增一个真实项目阶段，补充对应的复杂度类型、拆解方法、证据类型和可展示等级。

### 与 P0 其他层的关系 / Relation to Other P0 Layers

可连接 AI Product Lab Layer，用于未来 AI 工具、Agent 工作台和复杂工作流产品实验；也可连接 What-if Expression Layer，把复杂系统想象转化为可模拟场景。

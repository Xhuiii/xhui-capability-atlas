# Capability Cards v0 / 能力卡片 v0

本文档是 P0 `Personal Skill Simulator Atlas / 个人技能模拟器地图` 的第一批能力卡片草稿。

这些卡片用于把 Tina 已经证明过的能力、可迁移能力、潜在能力和未来可展示方向整理成结构化材料。当前版本只使用脱敏、抽象表达，不包含公司敏感信息、内部项目细节、同事姓名或内部数据来源。

这些卡片不是最终作品集文案，而是 P0 后续生成能力地图、项目卡、证据链、时间线和可视化展示的基础素材。

---

## Card 01 — 复杂系统体验拆解能力 / Complex System Experience Decomposition

### 能力名称 / Capability Name

复杂系统体验拆解能力 / Complex System Experience Decomposition

### 能力一句话定义 / One-line Definition

能够把复杂产品、复杂流程或多角色系统拆解成清晰的用户路径、功能模块、体验问题和设计机会。

### 适用场景 / Use Cases

- 多角色、多流程的产品体验分析
- 复杂业务工具的信息架构梳理
- 跨页面、跨状态的用户路径拆解
- 从模糊问题中识别关键体验断点
- 为后续交互设计、指标优化或产品迭代建立结构

### 已证明的证据类型 / Evidence Type

- 脱敏后的流程图 / Flow map
- 信息架构草图 / Information architecture
- 用户路径分析 / User journey analysis
- 问题清单与优先级判断 / Issue framing
- 设计复盘记录 / Design reflection

### 可能关联项目 / Related Projects

- P0：能力卡、项目卡、证据链、时间线
- P1：Agent 工作流、项目管理与任务拆解
- P2：AI 产品或复杂工具原型的信息架构

### 可迁移方向 / Transferable Directions

- B2B / SaaS 产品体验设计
- AI 工具工作流设计
- 后台系统与运营工具优化
- 产品策略与需求澄清
- 服务蓝图和系统体验诊断

### 可以如何在 P0 中展示 / How to Show in P0

- 展示为一张“复杂系统拆解”能力卡
- 配一张脱敏用户路径图或模块关系图
- 用 before / after 的方式展示问题如何被结构化
- 连接到相关项目卡和证据链

### 下一步最小行动 / Next Small Action

选择一个已脱敏的复杂流程，画出 1 张用户路径草图，并标出 3 个体验断点。

## Card 02 — 端到端交互设计交付能力 / End-to-end Interaction Design Delivery

### 能力名称 / Capability Name

端到端交互设计交付能力 / End-to-end Interaction Design Delivery

### 能力一句话定义 / One-line Definition

能够从问题理解、流程设计、界面方案、状态覆盖到交付沟通，完成一套可落地的交互设计过程。

### 适用场景 / Use Cases

- 从需求到方案的完整交互设计
- 多状态、多异常路径的界面设计
- 与产品、研发或其他角色协作推进设计落地
- 设计规范、组件、交互说明的整理
- 对已有体验进行迭代优化

### 已证明的证据类型 / Evidence Type

- 脱敏设计稿 / Sanitized design mockups
- 交互流程图 / Interaction flow
- 状态表 / State table
- 设计说明 / Design specification
- 版本迭代记录 / Iteration notes

### 可能关联项目 / Related Projects

- P0：项目卡、能力卡、交付证据链
- P1：任务拆解、执行闭环、diff 检查流程
- P2：AI 产品原型或工具界面

### 可迁移方向 / Transferable Directions

- 产品设计 / Product design
- 交互设计 / Interaction design
- 原型设计 / Prototyping
- Design system 协作
- AI-native 工具体验设计

### 可以如何在 P0 中展示 / How to Show in P0

- 用一张端到端交付路径展示从问题到方案的过程
- 展示脱敏后的关键界面、状态覆盖和交互逻辑
- 用“输入 - 过程 - 输出”的结构连接证据
- 标注该能力如何支持 P2 原型实验

### 下一步最小行动 / Next Small Action

选一个可脱敏的设计案例，列出它的 5 个交付环节：问题、流程、界面、状态、说明。

## Card 03 — 创新方案抽象与专利化能力 / Innovation Abstraction and Patent Framing

### 能力名称 / Capability Name

创新方案抽象与专利化能力 / Innovation Abstraction and Patent Framing

### 能力一句话定义 / One-line Definition

能够从具体产品问题或设计方案中抽象出可复用的创新机制、系统方法和可表达的发明点。

### 适用场景 / Use Cases

- 从具体方案中提炼创新点
- 把设计机制抽象成通用方法
- 整理专利方向或技术方案表达
- 梳理问题、方案、效果之间的逻辑链
- 将创意从界面层提升到系统机制层

### 已证明的证据类型 / Evidence Type

- 脱敏创新点说明 / Sanitized innovation notes
- 方案抽象图 / Mechanism diagram
- 问题-方案-效果链路 / Problem-solution-effect chain
- 专利化表达草稿 / Patent framing draft
- 设计决策记录 / Design rationale

### 可能关联项目 / Related Projects

- P0：能力卡、证据链、潜在能力展示
- P2：AI 产品机制、工具逻辑、系统方案
- P4：未来商业想象或可产品化方向

### 可迁移方向 / Transferable Directions

- 创新方案表达
- 产品机制设计
- 专利草案协作
- AI 产品概念定义
- 商业化产品想象

### 可以如何在 P0 中展示 / How to Show in P0

- 展示为“从具体方案到抽象机制”的能力路径
- 用脱敏图示表达创新点如何被提炼
- 连接到 What-if 表达卡，展示想象力如何转化为系统机制
- 标注可复用的方法框架

### 下一步最小行动 / Next Small Action

选择一个已脱敏的设计想法，写出它的“问题 - 机制 - 效果”三段式说明。

## Card 04 — 多端/多场景体验理解能力 / Multi-platform and Multi-context Experience Understanding

### 能力名称 / Capability Name

多端/多场景体验理解能力 / Multi-platform and Multi-context Experience Understanding

### 能力一句话定义 / One-line Definition

能够理解同一能力、流程或服务在不同终端、不同场景和不同用户状态下的体验差异。

### 适用场景 / Use Cases

- Web、mobile、desktop 或其他端的体验差异分析
- 多场景任务流程设计
- 不同使用频率、紧急程度、环境限制下的体验判断
- 跨设备的信息连续性设计
- 为复杂产品建立场景化体验地图

### 已证明的证据类型 / Evidence Type

- 多端体验对比 / Cross-platform comparison
- 场景地图 / Context map
- 任务流程分析 / Task flow analysis
- 端侧差异说明 / Platform difference notes
- 脱敏设计方案 / Sanitized design solution

### 可能关联项目 / Related Projects

- P0：多场景能力卡和项目卡
- P2：多端 AI 工具、互动产品或原型实验
- P3：内容在不同平台的表达方式
- P5：不同状态下的行动和自我记录场景

### 可迁移方向 / Transferable Directions

- 跨平台产品设计
- 移动端和桌面端体验设计
- 场景化服务设计
- AI companion 或 AI 工具多端体验
- 内容产品与创作工具设计

### 可以如何在 P0 中展示 / How to Show in P0

- 用一张多端场景矩阵展示不同端、不同场景的体验差异
- 将同一用户任务拆成多个使用环境
- 展示 Tina 如何判断哪些体验应该一致，哪些应该因场景变化
- 连接到未来 P2 产品实验

### 下一步最小行动 / Next Small Action

选一个常见任务，分别写出它在 mobile、desktop 和低注意力场景下的体验差异。

## Card 05 — 用户洞察与体验指标转化能力 / User Insight and Experience Metric Translation

### 能力名称 / Capability Name

用户洞察与体验指标转化能力 / User Insight and Experience Metric Translation

能够把用户观察、体验问题和模糊反馈转化为可讨论、可排序、可验证的体验指标或设计判断。

- 定性观察到设计机会的转化
- 体验指标定义
- 设计方案优先级判断
- 迭代效果复盘

### 已证明的证据类型 / Evidence Type

- 脱敏用户反馈摘要 / Sanitized feedback summary
- 洞察归类表 / Insight clustering
- 体验问题优先级 / Experience issue priority
- 指标定义草稿 / Metric definition draft
- 迭代复盘 / Iteration review

### 可能关联项目 / Related Projects

- P0：证据链、能力卡、项目复盘
- P1：Issue 分类、任务优先级和复盘机制
- P2：AI 产品实验的验证指标
- P3：内容反馈与受众观察

### 可迁移方向 / Transferable Directions

- 用户研究协作
- 体验策略
- 产品迭代评估
- AI 产品验证
- 内容与个人品牌反馈分析

### 可以如何在 P0 中展示 / How to Show in P0

- 展示从“用户反馈”到“体验指标”的转化链路
- 用脱敏案例说明如何聚类问题和判断优先级

- 作为 P0 中“证据如何支持能力”的代表卡片

### 下一步最小行动 / Next Small Action

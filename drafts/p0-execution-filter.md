# P0 Execution Filter / P0 执行筛选机制

本文档是 P0 `Personal Skill Simulator Atlas / 个人技能模拟器地图` 的想法筛选草稿，用来判断一个想法现在应该停留在信号层、进入模拟层，还是推进到执行层。

这个机制的目的不是压制想象力，而是避免每个灵感都立刻变成任务。P0 需要同时保护灵感、保留可能性，并把真正适合推进的内容变成具体产物。

---

## 1. Signal / 信号层

Signal / 信号层指的是：有趣、有感觉、可能重要，但暂时不执行的想法。

这一层适合保存那些还没有清晰形态，但明显触动了观察、情绪、审美、产品直觉或未来想象的内容。

### 判断标准

- 有感觉，但还说不清楚为什么
- 有画面感、情绪感或隐喻感
- 暂时没有明确用户、场景或产物
- 可能属于 P0-P5，但还不确定
- 现在执行会过早、过重或分散主线

### 可以记录什么

- 一句话灵感
- 关键词
- 观察片段
- 情绪描述
- 视觉隐喻
- 暂存标签

### 不需要做什么

- 不需要立刻开 issue
- 不需要立刻做原型
- 不需要立刻判断商业价值
- 不需要立刻写成完整卡片

### 示例

“如果记忆像天气一样，有晴天记忆、暴雨记忆、潮湿记忆。”

当前处理方式：保留为 Signal / 信号，不急着执行。

---

## 2. Simulation / 模拟层

Simulation / 模拟层指的是：想法已经可以作为 What-if 表达卡、能力模拟或概念推演，但还不一定要做成真实产品。

这一层是 P0 的核心。它让想法从“有感觉”进入“可观察、可拆解、可显影技能”的状态。

### 判断标准

- 已经有清晰的 What-if 设定
- 能说明观察对象
- 能推导隐性需求
- 能想象一种或多种可能形态
- 能看出它显影了哪些潜在技能
- 可以关联 P0-P5 中的一个或多个项目域

### 可以产出什么

- What-if Expression Card / What-if 表达卡
- Capability Simulation / 能力模拟
- User Scenario / 用户场景
- Service Idea / 服务想法
- Interface Sketch / 界面草图
- Content Direction / 内容方向

### 不需要做什么

- 不一定要立刻做完整产品
- 不一定要开开发任务
- 不一定要验证所有可行性
- 不一定要进入公开作品集

### 示例

“如果城市可以感知孤独，它会如何通过空间、灯光、座椅和低负担服务回应人？”

当前处理方式：进入 Simulation / 模拟层，写成 What-if 表达卡。

---

## 3. Execution / 执行层

Execution / 执行层指的是：想法已经足够清楚，可以变成 issue、草稿、原型、工具、内容或其他具体产物。

这一层需要更强的边界感。进入执行层的内容应该小、清楚、可检查。

### 判断标准

- 有明确产物
- 有明确下一步
- 可以在一次小任务中推进
- 可以通过 diff、截图、文档或原型检查
- 不包含敏感信息
- 符合当前优先级

### 可以产出什么

- GitHub Issue / GitHub 任务
- Draft / 草稿
- Prototype / 原型
- Tool / 工具
- Data Structure / 数据结构
- Visual Concept / 视觉概念
- Content Piece / 内容稿
- Update Log Entry / 更新记录条目

### 执行前检查

- 是否真的需要现在做？
- 是否能被拆成更小一步？
- 是否只影响少量文件？
- 是否会误触 README 基线规则？
- 是否可能写入敏感信息？
- 是否需要用户先确认范围？

### 示例

“创建 `drafts/what-if-expression-cards-v0.md`，写 5 张 What-if 表达卡样例。”

当前处理方式：进入 Execution / 执行层，生成具体草稿文件。

---

## 4. Three-layer Flow / 三层流转

一个想法不需要一开始就知道终点。它可以这样流转：

```text
Signal / 信号层
  -> Simulation / 模拟层
  -> Execution / 执行层
```

也可以停留、回退或被暂存：

- Signal 可以长期保留，不必执行
- Simulation 可以只作为技能显影，不必产品化
- Execution 可以拆回更小的 Simulation
- 不安全或未脱敏的内容不进入任何公开或仓库记录

---

## 5. Quick Filter Questions / 快速筛选问题

当一个新想法出现时，可以用这 5 个问题快速判断层级：

1. 它现在只是有感觉，还是已经能说清楚设定？
2. 它能显影 Tina 的某种潜在技能吗？
3. 它是否能关联 P0-P5 中的某个项目域？
4. 它是否已经有一个小到可以执行的下一步？
5. 它是否安全、脱敏、适合写进仓库？

如果只有感觉，放在 Signal / 信号层。

如果能显影技能，放在 Simulation / 模拟层。

如果有清楚产物和下一步，放在 Execution / 执行层。

---

## 6. Core Principle / 核心原则

P0 不需要把每个想法都变成任务。

P0 需要做的是：

- 保存信号 / Preserve signals
- 模拟能力 / Simulate capabilities
- 筛选执行 / Filter execution
- 沉淀资产 / Accumulate assets

这样，想象力不会被任务化压扁，任务也不会被无边界的想象拖散。
